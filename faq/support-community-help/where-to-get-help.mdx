# Where do I get additional support or community help?

This page guides you on how to obtain official support, find community resources, and engage with the maintainers and contributors of The System Design Primer project.

---

## Official Support Channels

- **GitHub Issues Tracker**
  - The primary place for submitting questions, reporting bugs, and engaging in discussions.
  - Search existing issues before opening a new one to find answers or ongoing conversations.
  - Use clear titles and detailed descriptions to help maintainers and contributors assist effectively.

- **Pull Requests**
  - Suggest improvements, fixes, or additions by submitting pull requests.
  - Follow the contribution guidelines for best practices on how to contribute.

## Community Resources

- **Discussion Forums & Q&A**
  - Although there is no official dedicated forum, many users engage through GitHub discussions in issues or pull requests.

- **Translation Collaboration**
  - The primer is available in many languages.
  - For help with translations or to contribute, please refer to the [translation guidelines](https://github.com/donnemartin/system-design-primer/issues/28).

- **Contribution Guidelines**
  - Review the [CONTRIBUTING.md](https://github.com/donnemartin/system-design-primer/blob/main/CONTRIBUTING.md) file for information on how to report issues effectively or contribute new content.

## Tips for Effective Support Requests

- Provide relevant details such as:
  - Description of the problem or question
  - Steps to reproduce issues (if any)
  - Environment or setup information
  - Links to related resources or files
- Be clear and concise.
- Search existing issues or documentation first to avoid duplication.
